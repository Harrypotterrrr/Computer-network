**Linux下的静态编译** 1651574 贾昊霖 # Linux 下的动态编译 ## 静态编译与动态编译的比较 - 动态编译：可执行文件需要附带一个的动态链接库，在执行时，需要调用其对应动态链接库中的命令。所以其优点一方面是缩小了执行文件本身的体积，另一方面是加快了编译速度，节省了系统资源。缺点一是哪怕是很简单的程序，只用到了链接库中的一两条命令，也需要附带一个相对庞大的链接库；二是如果其他计算机上没有安装对应的运行库，则用动态编译的可执行文件就不能运行 - 静态编译：编译器在编译可执行文件的时候，将可执行文件需要调用的对应动态链接库(.so)中的部分提取出来，链接到可执行文件中去，使可执行文件在运行的时候不依赖于动态链接库 ## 动态编译 - printf("hello，world") - 用`gcc -o test test.cpp`指令编译 - 用`wc test` 观察执行文件的字节数 ![](./pic/dy_st_init-1-1.png) 三个数分别为 行数，词数，字节数 - 在[Mysql官网](https://dev.mysql.com/downloads/connector/cpp/)下载mysql的链接libmysqlclient，版本为8.0.12 下载的压缩包名称为：`mysql-connector-c++-8.0.12-linux-glibc2.12-x86-64bit.tar.gz` - 通过tar -xzvf mysql-connector-c++-8.0.12-linux-glibc2.12-x86-64bit.tar.gz` 解压该压缩包，注意其中z为`.gz`格式 ![](./pic/dy_st_init-2-7.png) - 备份之前的mysql目录以及源代码 注意**一定要拷贝**，原因是下载的`connector`里不存在`mysql_config`等可执行文件 ![](./pic/dy_st_init-2-8.png width=420) - 将压缩包中的对应内容全部拷贝到之前建立的mysql文件夹中 ~~~~~~~~~~~~ bush cp -r include/* /usr/include/mysql cp -r lib64/* /usr/lib64/mysql ~~~~~~~~~~~~ ![](./pic/dy_st_init-2-9.png) 如果遇到覆盖，选择`y` - 观察拷贝结果 ![](./pic/dy_st_init-2-10.png) `libmysqlclient.a`为所需的静态链接库文件，观察lib64目录，mysql_config等可执行文件正常 （之后可能需要reboot或者restart service） - 进入demo目录，通过下述指令，对`demo.cpp`静态编译` ~~~~~~~~~~~~~ bush g++ -o out2 demo.cpp -static -I/usr/include/mysql -L/usr/lib64/mysql -lmysqlclient -lpthread -ldl ~~~~~~~~~~~~~ ![](./pic/dy_st_init-2-11.png) - 其中 后三个选项极为重要， `-I/usr/include/mysql`可以替换成在动态编译时用到的`$(mysql_config --cflags)`， 而`-L/usr/lib64/mysql -lmysqlclient -lpthread -ldl` 不能替换成`$(mysql_config --libs)` 原因在于，有的库编译器找不到 ![](./pic/dy_st_init-2-13.png) - 运行out2并与动态编译的可执行文件out对比大小 ![](./pic/dy_st_init-2-12.png) 均可正常执行，对比大小发现，动态编译的可执行文件大约在10k左右，而静态编译的可执行文件约在10M左右 - 检验生成地`out2`为静态可执行文件 ![](./pic/dy_st_init-2-16.png) - makefile的编写 ~~~~~~~~~~~ makefile linenumbers # s指static，d指dynamic CFLAGS_s = -static $$(mysql_config --cflags) -L/usr/lib64/mysql -lmysqlclient -lpthread -ldl CFLAGS_d = $$(mysql_config --cflags --libs) CC = g++ FILE = mysql_demo # 此为静态编译，生成mysql_demo $(FILE): $(FILE).cpp $(CC) -o $(FILE) $(FILE).cpp $(CFLAGS_s) .PHONY = clean $(FILE)_d # 此为动态编译，生成mysql_demo_d $(FILE)_d: $(FILE).cpp $(CC) -o $(FILE)_d $(FILE).cpp $(CFLAGS_d) clean: rm -f $(FILE) $(FILE)_d ~~~~~~~~~~ 注意这里多加了一个伪目标 便于生成动态与静态可执行文件，运行完全正确！ ![](./pic/dy_st_init-2-17.png)
